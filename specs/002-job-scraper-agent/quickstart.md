# Quickstart: AI-Powered Job Scraper Agent

## Prerequisites

- .NET 10.0 LTS SDK installed
- Local OpenAI-compatible LLM running at http://192.168.1.21:8080/v1
- Internet connection to reach Indeed.com
- A resume file (plain text or markdown format)

## Configuration

### 1. LLM Endpoint Configuration

Set the LLM endpoint in `src/Fidalgo.Agent/appsettings.json`:

```json
{
  "LLM": {
    "Endpoint": "http://192.168.1.21:8080/v1",
    "Model": "local-model",
    "ApiKey": "local"
  },
  "DatabasePath": "jobs.db"
}
```

### 2. Running the Agent

```bash
# Navigate to project root
cd src/Fidalgo.Agent

# Basic usage with email, keywords, and resume
dotnet run -- --email "user@example.com" --keywords "software engineer" --resume "./resume.md"

# With all options (including optional career narrative)
dotnet run -- --email "user@example.com" --keywords "software engineer" --resume "./resume.md" --narrative "./career-narrative.md"

# Multiple keywords
dotnet run -- --email "user@example.com" --keywords "software engineer, cloud architecture" --resume "./resume.md"
```

### 3. Command-Line Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| --email | Yes | User's email address |
| --keywords | Yes | Comma-separated job search keywords |
| --resume | Yes | Path to resume file (plain text or markdown) |
| --narrative | No | Path to optional career narrative file |

## What Happens

1. The tool validates all inputs (email format, file existence)
2. Connects to the local LLM endpoint
3. Creates an AI agent with the configured tools (fetch, save_job, get_jobs)
4. The agent fetches Indeed.com search results for the given keywords
5. For each job posting, the agent:
   - Fetches the full job description
   - Checks the database to avoid duplicates
   - Compares the job against the user's resume and narrative
   - Scores the match (0-100%)
   - Saves the job with pros, cons, resume hints, and recommendation
6. After processing up to 100 pages or exhausting results, the agent stops

## Querying Stored Jobs

```bash
# Query all jobs for a user
dotnet run -- --email "user@example.com" --query-jobs

# Query with filters (coming in future releases)
dotnet run -- --email "user@example.com" --query-jobs --employer "Microsoft" --date-from "2026-01-01"
```

## Troubleshooting

| Problem | Likely Cause | Solution |
|---------|-------------|----------|
| "LLM endpoint unreachable" | LLM server not running | Start the LLM server at http://192.168.1.21:8080/v1 |
| "Resume file not found" | Incorrect path | Verify the file path is correct and accessible |
| "Invalid email format" | Malformed email | Ensure email is valid (e.g., user@example.com) |
| "No jobs found" | Indeed returned no results | Try different keywords or run again to retry search |
